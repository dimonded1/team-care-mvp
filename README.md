# Команда заботы — MVP

Отдельный greenfield-проект мобильного интерактива программы «Опека» фонда НИКА. Старый событийный прототип не используется как техническая или визуальная основа; перенесены только проверенные фотографии животных как контент.

## Сценарий

1. Загрузка.
2. Стартовый экран.
3. 11 свайп-вопросов: вправо/«Да», влево/«Нет».
4. Мэтч и орбита одного животного.
5. Короткое визуальное объяснение: подопечный — Солнце, опекуны — планеты и защитный круг.
6. Четыре последовательных модуля опеки: знакомство, визит, прогулка, внимание к здоровью.
7. Объяснение опеки и команды до пяти опекунов.
8. Финал, CTA и PNG-карточка для сохранения/шеринга.

В клиентском MVP нет backend и активной связи с Битрикс24. Локально сохраняется только анонимный прогресс. Выключенная серверная заготовка интеграции и все точки настройки находятся в [`integrations/bitrix`](integrations/bitrix/README.md) и [`.env.bitrix.example`](.env.bitrix.example).

## Локальный запуск

```bash
npm ci
npm run dev
```

## Проверки

```bash
npm run test
npm run build
node --test integrations/bitrix/bitrix.node-test.mjs
```

После запуска Docker полный мобильный smoke-тест (реальный свайп, 11 вопросов, четыре ситуации и генерация карточки) выполняется так:

```bash
npm run smoke
```

Скриншоты проверки сохраняются локально в `artifacts/` и не коммитятся.

## Docker

```bash
docker compose build
docker compose up -d
```

Приложение будет доступно на `http://localhost:8080`. Runtime использует non-root nginx, read-only filesystem, healthcheck и security headers.
